# Founder Preferences

## Time Management
- Client work gets mornings (peak focus)
- Biz dev and content in afternoons
- Never schedule meetings before 10am
- Batch similar tasks (all emails at once, all code at once)

## Communication
- Direct and action-oriented
- Tables for status, bullets for action items
- Always include deadlines in recommendations
- Flag revenue impact of decisions

## Delegation
- Delegate: Design, routine testing, content scheduling
- Keep: Architecture decisions, client relationships, strategy
- Outsource threshold: if it takes > 2 hours and isn't core value

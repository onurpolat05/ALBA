# Memory System Guide

## Structure

```
memory/
├── state/dashboard.md    # Content calendar, upcoming posts, ideas backlog
├── state/todo.md         # Active content tasks
├── knowledge/learnings.md # What content works, audience insights
├── knowledge/preferences.md # Voice guidelines, platform preferences
├── knowledge/errors.md   # Research dead-ends, publishing issues
├── projects/[series]/    # Per content series/campaign
└── daily/YYYY-MM-DD.md   # Session logs with content produced
```

## Creator-Specific Usage

- Dashboard: content calendar view with deadlines per platform
- Ideas backlog: capture ideas during research for future content
- Learnings: track what content performs well and why
- Projects: organize long-form series or campaigns

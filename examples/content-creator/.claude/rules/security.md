# Security Rules

## Secrets Protection
- Never commit API keys or tokens
- Never expose analytics credentials
- Use .env.example with placeholders

## Content Safety
- Verify facts and statistics before including in content
- Don't plagiarize - always rephrase and cite
- Check image/media licensing before recommending
- Be careful with user data in case studies

## Safe Operations
### Always Ask Before
- Publishing or scheduling content
- Sharing drafts with external parties
- Using real names/data in examples
- Bulk operations on content calendar

### Safe Without Asking
- Researching topics
- Drafting outlines
- Updating local memory files
- Analyzing content ideas

# Security Rules

## Secrets Protection
- Never commit .env, API keys, or tokens
- Never output credentials in status reports
- Use .env.example with placeholders

## Safe Operations
### Always Ask Before
- Sending emails or messages to team/clients
- Creating or modifying Trello/Notion items
- Sharing project data externally
- Bulk operations on tasks

### Safe Without Asking
- Reading project status and tasks
- Generating internal reports
- Searching for information
- Updating local memory files

## Data Sensitivity
- Client data stays in project folders only
- Don't mix client information across projects
- Anonymize when creating examples or templates

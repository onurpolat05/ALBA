# User Preferences

Working style and preferences for the agent.

---

## Work Style

- **Organized and structured**: Clear processes, defined workflows, documented decisions
- **Deadline-driven**: Every task has an owner and a due date
- **Team-first**: Focus on unblocking others, removing obstacles
- **Data-informed**: Use metrics (velocity, burndown, cycle time) for decisions
- **Proactive communication**: Stakeholders should never be surprised

---

## Communication Preferences

| Preference | Detail |
|------------|--------|
| Language | English |
| Format | Tables for status tracking, bullets for action items |
| Length | Professional and concise, skip filler |
| Tone | Professional, clear, direct |
| Meeting notes | Decisions / Action Items / Follow-ups format |
| Status updates | Project > Status > Risks > Next Steps format |

---

## Output Formats

### Status Update Template
```
**[Project Name] - Weekly Update**
- Status: [On Track / At Risk / Blocked]
- Completed: [bullet list]
- In Progress: [bullet list]
- Risks: [bullet list with mitigation]
- Next Week: [bullet list]
```

### Meeting Notes Template
```
**[Meeting Name] - [Date]**
Attendees: [list]

Decisions:
- [Decision 1]

Action Items:
- [ ] [Action] - Owner - Due date

Follow-ups:
- [Item to revisit]
```

### Escalation Template
```
**Issue:** [one-line description]
**Impact:** [what's affected]
**Current Status:** [what we know]
**Ask:** [what we need]
**Deadline:** [when we need it by]
```

## Sensitivities

### DON'T
- Send client communications without explicit approval
- Change project timelines without discussion
- Make resource allocation decisions unilaterally
- Skip stakeholder updates even when things are on track
- Use informal language in client-facing content

### DO
- Flag risks early with proposed mitigations
- Track action items with owners and due dates
- Provide options when decisions are needed
- Keep meeting notes structured and actionable
- Remind about deadlines 2 days in advance

---

## Work Hours

- Core hours: 9 AM - 6 PM, Monday-Friday
- Stand-ups at 9:00 AM daily
- No-meeting blocks: Thursday afternoon (focus time)
- Deadlines are non-negotiable unless escalated and re-negotiated

---

## Priority Framework

1. **Client commitments**: Contractual deadlines, deliverables
2. **Team blockers**: Anything preventing team members from working
3. **Sprint goals**: Current sprint deliverables
4. **Planning**: Next sprint/quarter preparation
5. **Process improvement**: Retro items, workflow optimization

## Decision Making

- For project decisions: Present options with pros/cons/risks
- For scheduling: Suggest the best option, explain conflicts
- For escalations: Draft the communication, wait for approval
- For routine updates: Proceed and inform
- Always include "recommended" option when presenting choices
